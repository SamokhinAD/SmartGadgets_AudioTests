Этот репозиторий содержит набор Python- и Bash-скриптов, с помощью которых демонстрируются некоторые мои технические навыки, необходимые в области тестирования умных аудиоустройств и голосовых ассистентов (например, Алисы).
Проекты демонстрируют базовое умение работать с аудиоданными и оборудованием, автоматизацией настройки тестовой среды и способами модульного тестирования.

Python tools:

    1. Acoustic_Data_Analyzer.py
    Скрипт для базового анализа сырых аудиофайлов (wav), имитирует начальный этап оценки акустических измерений:
        1) чтение wav, конвертация и нормализация данных, расчет RMS по громкости;
        2) визуализация: построение временного графика сигнала для визуальной оценки качества и структуры сигнала.
    1.1. test_Alice_KAPI.wav - дополнительный файл для тестирования скрипта. В нем представлен отрывок речи Алисы.

    2. Auto_Environ_Setup.py
    Скрипт для автоматизации подготовки некоторых частей тестовой среды:
        1) использование ping для проверки доступности тестового устройства по IP-адресу - с автоматической адаптацией синтаксиса команды под Linux/Windows;
        2) удаление всех старых лог-файлов из указанной папки (с сохранением директорий) - автоматизация очистки места;
        3) вывод консольного отчета о готовности среды к работе.
    2.1. test_logs - дополнительная директория для проверки работы скрипта по удалению старых лог-файлов. В нем представлены три txt-файла и одна директива, не несущие смысловой нагрузки.

    3. Acoustic_Test_Runner.py
    Скрипт запуска тестов с использованием стандартного фреймворка unittest:
        1) загрузка параметров теста (уровень шума, тип устройства) из указанного файла json только один раз перед запуском всех тестов;
        2) выполнение методов (test_...), которые имитируют оценку таких алгоритмов как:
            а) проверка точности распознавания речи (ASR) при шуме (assertGreater);
            б) проверка уровня громкости умного устройства на расстоянии (assertLess);
            в) использование метода setUp для имитации подготовки перед каждым тестом.
    3.1. test_acoustic.json - дополнительный файл для проверки работы скрипта с форматом json. В нем представлен краткий пример возможных данных для обработки.

Bash tools:

    1. mic_test.sh
        Скрипт для записи аудиосигнала с микрофона через утилиту arecord в течение 5 секунд. Результат сохраняется в виде wav-файла.

    2. audio_cleaner.sh
        Скрипт для удаления тех wav-файлов (старых записей) из указанной директории, для которых срок последнего изменения больше семи дней.

Навыки, продемонстрированные в репозитории:

    - Python: unittest, NumPy, SciPy и др. библиотеки
    - Базовые навыки Bash-скриптинга
    - Работа с автоматизацией и скриптами
    - Работа с файлами и конфигурациями (json, логи)
    - Работа с акустическими метриками (RMS, ASR)
